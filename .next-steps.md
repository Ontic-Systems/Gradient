This file tracks what to work on next for the Gradient programming language.

### ✅ COMPLETED - Self-Hosting Compiler v1.0 + Bootstrap Infrastructure

**🎉 ALL PHASES COMPLETE! SELF-HOSTING + BOOTSTRAP INFRASTRUCTURE! 🎉**

The Gradient self-hosting compiler is fully implemented with complete bootstrap infrastructure:

### Core Compiler Components (DONE)

1. **Token Module** - DONE
   - Token enum with all token kinds
   - 18 type checker tests

2. **Lexer** - DONE
   - Source code tokenization
   - Full Gradient syntax support

3. **Parser** - DONE
   - Recursive descent parser
   - Complete Gradient grammar

4. **Type System Module** (`compiler/types.gr`) - DONE
   - Ty enum with all type variants
   - Capability enum for memory safety
   - EffectSet for effect tracking
   - Type classification and subtyping
   - 15 type checker tests

5. **Type Checker Module** (`compiler/checker.gr`) - DONE
   - TypeError enum with 15+ error variants
   - Hindley-Milner unification
   - Scope management
   - Type environment with substitutions
   - 18 type checker tests

6. **IR Module** (`compiler/ir.gr`) - DONE
   - SSA-based intermediate representation
   - 40+ instruction variants
   - BasicBlock control flow
   - IrFunction and IrModule structures
   - 43 IR tests

7. **IR Builder Module** (`compiler/ir_builder.gr`) - DONE
   - Incremental IR construction
   - Instruction emission helpers
   - Expression and statement lowering
   - Local variable management
   - 14 IR builder tests

8. **Compiler Driver** (`compiler/compiler.gr`) - DONE
   - Main compilation pipeline (5 phases)
   - compile_file() and compile_string() entry points
   - CompilerConfig for optimization/debug settings
   - Phase-by-phase execution with progress tracking
   - 10 compiler integration tests

### Bootstrap Infrastructure (DONE)

9. **Bootstrap Module** (`compiler/bootstrap.gr`) - DONE
   - BootstrapConfig for bootstrap settings
   - BootstrapState tracking all phases
   - 5-phase bootstrap process:
     1. NotStarted
     2. CompilingModules (compile all 5 modules)
     3. ValidatingOutput (compare to reference)
     4. SelfCompiling (build final binary)
     5. Complete/Failed
   - ModuleCompileResult per-module tracking
   - ValidationResult with line-by-line diff
   - OutputDifference types (MissingLine, ExtraLine, ChangedLine)
   - run_bootstrap() orchestration function
   - 9 bootstrap tests
